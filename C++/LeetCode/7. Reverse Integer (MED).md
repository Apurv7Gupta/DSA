Given a signed 32-bit integer `x`, return `x` _with its digits reversed_. If reversing `x` causes the value to go outside the signed 32-bit integer range `[-231, 231 - 1]`, then return `0`.

**Assume the environment does not allow you to store 64-bit integers (signed or unsigned).**

Solution:

- Take the given number, lets call it "x"
- Divide it by 10 and get the remainder into a new variable, "Digit".
- Do this until the given number is zero.
- Define another variable "ans",  that will store these separated end digits and form the final reversed number.
- Initialize it with 0 outside the loop.
	- inside the loop, after getting the end digit in the "ans" variable, multiply it with 10 to get to the next place value for adding another digit, and then add the digit variable with it.
	- Then divide the given number, "x" by 10, to remove the last digit from the original, as it is already processed.

Additional Constraint:
multiplying "ans" by 10 can reach a place value greater than INT_MAX, or if the given value is a negative integer, "ans" can reach below INT_MIN

To address this:
	- add an if block before the "ans * 10" statement is hit
	- But, you cannot write "if (ans * 10 > INT_MAX || ans * 10 < INT_MIN)", as that will hit the "ansx10" line first
	- So instead, you decrease the value of INT_MAX `{By doing INT_MAX/10}` to PRECHECK if ans is going to overflow if it is multiplied by 10

This way, you can avoid overflow while still checking for it.


```cpp

class Solution {

public:

    int reverse(int x) {

          int ans = 0;

    while (x != 0)

    {

  

        int digit = x % 10;

	    if ((ans > INT_MAX / 10) || (ans < INT_MIN / 10)) { return  0;}

        ans = (ans * 10) +  digit;

        x = x / 10;

    }
    return ans;

    }

};

```
